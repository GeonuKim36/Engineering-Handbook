# Writing Framework

## Purpose

The Engineering Handbook is organized around engineering questions rather than document categories.

Its goal is to help readers understand not only what a concept is, but also why it exists, how it works, when it should be used, and where it is applied.

## Core Questions

Every document should be written to answer five questions:

### What is it?
Define the concept clearly.

Typical content:
- Overview
- Definition
- Terminology
- Symbols and notation
- Key characteristics

### Why does it exist?
Explain the motivation and problem context.

Typical content:
- Background
- Motivation
- Problem statement
- Historical context
- Limitations of previous approaches

### How does it work?
Explain the underlying principle or procedure.

Typical content:
- Theory
- Operating principle
- Mathematical derivation
- Signal model
- System model
- Algorithm
- Implementation
- Performance analysis

### When should it be used?
Explain the conditions under which the concept is appropriate.

Typical content:
- Selection criteria
- Design trade-offs
- Suitable scenarios
- Unsuitable scenarios
- Assumptions
- Constraints

### Where is it applied?
Explain practical usage and related domains.

Typical content:
- Applications
- Practical examples
- Related technologies
- Research trends

## Content Blocks

The following content blocks may be reused across documents as needed.

- Overview
- Definition
- Terminology
- Background
- Motivation
- Problem Statement
- History
- Theory
- Physical Principle
- Operating Principle
- Mathematical Description
- Mathematical Derivation
- Signal Model
- System Model
- Algorithm
- Implementation
- Design Considerations
- Complexity
- Performance
- Advantages
- Limitations
- Applications
- Examples
- Related Topics
- References
- Further Reading

## Writing Style

- Explain concepts from first principles.
- Build intuition before presenting equations.
- Use equations to support understanding, not replace it.
- Prefer clear technical language over informal wording.
- Link related topics so each document becomes part of a connected knowledge graph.

## Notes

This framework is intentionally flexible. Not every document needs every block. The right blocks should be selected according to the topic being written.
